# 🛠️ 문제 해결 (Troubleshooting)

이 문서는 '큐브 마법사' 앱을 사용하면서 발생할 수 있는 주요 문제와 해결 방법을 다룹니다. 특히 태블릿 및 모바일 기기에서의 원활한 동작을 위한 가이드를 포함합니다.

## 1. 카메라 화면이 나오지 않아요 (흰 화면)

### 보안 컨텍스트 (HTTPS) 필수
대부분의 현대식 브라우저(iOS Safari, Chrome, Samsung Internet 등)는 보안 연결(**HTTPS**)이 아닌 경우 카메라 접근을 엄격히 차단합니다.
-   **확인 방법**: 주소창의 URL이 `https://`로 시작하는지 확인하세요.
-   **예외**: `http://localhost`는 개발 목적으로 예외 허용되지만, IP 주소(예: `http://192.168.0.5:5173`)로 접속할 때는 반드시 HTTPS 설정을 해야 합니다.

### 카메라 로딩 및 솔버 예열 (태블릿 필수 사항)
일부 태블릿이나 저사양 기기에서는 '시작하기' 클릭 직후 약 0.5~2초 정도 일시적인 멈춤 현상이 발생할 수 있습니다.
-   **현상**: 버튼 클릭 후 '화면 전환'은 즉시 일어나지만, 카메라 화면이 나오기까지 약간의 지연이 발생함.
-   **이유**: 큐브 솔버(Solver) 라이브러리가 정답을 찾기 위한 거대 테이블을 생성(약 5초 소요)하는 동안 UI가 멈추지 않도록, 앱은 의도적으로 **화면 전환 후 0.5초 뒤에 초기화**를 시작합니다.
-   **해결**: 이는 기기 성능에 따른 정상적인 동작입니다. 잠시만 기다려 주시면 카메라 화면이 활성화됩니다. 만약 10초 이상 아무 반응이 없다면 브라우저를 새로고침해 주세요.

### 고해상도 카메라 지원
최신 태블릿의 고해상도 카메라 사용 시 초기 그리드 설정이 어긋날 수 있습니다.
-   **해결**: 화면 네 모서리에 있는 흰색 동그라미(핸들)를 드래그하여 큐브 크기에 맞게 직접 조정할 수 있습니다. 한 번 조정하면 해당 세션 동안 유지됩니다.

### 카메라 권한 허용
처음 앱을 실행할 때 카메라 권한 요청 팝업에서 **[허용]**을 눌러야 합니다.
-   **iOS**: 설정 > Safari > 카메라 > [확인 또는 허용]
-   **Android Chrome**: 주소창 왼쪽의 자물쇠 아이콘 > 사이트 설정 > 카메라 > [허용]


## 2. 실시간 자동 정렬이 안 돼요

### OpenCV.js 로딩 대기
앱 시작 시 약 5~10MB 크기의 OpenCV.js 라이브러리를 CDN에서 내려받습니다. 네트워크 속도에 따라 몇 초 정도 시간이 걸릴 수 있습니다.
-   **해결**: 디버그 모드에서 `OpenCV: Ready`가 뜰 때까지 잠시 기다려 주세요.

### MediaPipe Hands 동작 환경
실시간 손 추적 기술은 어느 정도의 CPU/GPU 성능을 요구합니다.
-   **해결**: 아주 오래된 태블릿(예: iPad Air 1세대 등)에서는 실시간 추적이 느릴 수 있습니다. 이 경우 '자동 정렬 끄기'를 누르고 수동으로 그리드를 맞춰주세요.

## 3. 색상 인식이 정확하지 않아요

### 조명 환경 개선
-   **너무 밝거나 어두운 곳**: 직사광선 아래나 너무 어두운 실내에서는 색상 왜곡이 심합니다. 밝고 균일한 실내 조명을 권장합니다.
-   **그늘짐**: 손이나 기기로 인해 큐브 면에 그늘이 지지 않도록 주의하세요.

### 사용자 캘리브레이션 (추천)
기기마다 카메라의 색감 특성이 다릅니다. 설정의 **[캘리브레이션]** 기능을 통해 현재 조명 환경에 맞춰 색상 기준점을 잡으면 정확도가 획기적으로 향상됩니다.

## 4. 구형 기기 호환성 (Legacy Support)

본 앱은 다음 기능을 통해 구형 기기를 지원합니다:
-   **aspect-ratio Fallback**: `aspect-ratio` CSS 속성을 지원하지 않는 브라우저에서도 카메라 화면이 깨지지 않도록 패딩 기반 레이아웃을 제공합니다.
-   **MediaPipe Error Handling**: 손 인식 모델 초기화 실패 시 전체 앱이 멈추지 않고 수동 캡처 모드로 전환됩니다.
