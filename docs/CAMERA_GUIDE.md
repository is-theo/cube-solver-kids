# 📸 카메라 조작 가이드 (Camera Guide)

본 프로젝트는 다양한 장치(스마트폰, 태블릿, 노트북 등)에서 최적의 큐브 인식을 지원하기 위해 유연한 카메라 제어 기능을 제공합니다.

## 1. 카메라 전환 (Facing Mode Switch)

태블릿(iPad 등)이나 최신 스마트폰에서는 전면 카메라와 후면 카메라를 모두 사용할 수 있습니다.

-   **후면 카메라 (Environment)**: 기본값입니다. 큐브를 직접 비추어 인식하기에 가장 적합합니다.
-   **전면 카메라 (User)**: 셀카 모드입니다. 태블릿을 거치대에 두고 사용할 때 유용할 수 있습니다.

### 사용 방법
카메라 화면 하단의 **[전면 카메라로]** 또는 **[후면 카메라로]** 버튼을 눌러 실시간으로 전환할 수 있습니다. 전환 시 기존 스트림은 안전하게 종료되고 새로운 설정으로 재시작됩니다.

## 2. 해상도 최적화

`useCamera` 훅은 다음과 같은 설정을 통해 최적의 해상도를 시도합니다:
-   **Ideal Resolution**: 1280x720 (720p)
-   **Aspect Ratio**: 브라우저와 장치가 지원하는 최적의 비율을 사용합니다.

## 3. 카메라 잠금 (Focus & Exposure Lock)

안정적인 색상 인식을 위해, 캡처 카운트다운이 시작되면 카메라의 **노출(Exposure)**과 **화이트 밸런스(White Balance)**를 고정(Lock)하려고 시도합니다.

-   이 기능은 브라우저와 하드웨어가 `applyConstraints` 및 관련 기능을 지원하는 경우에만 작동합니다 (주로 안드로이드 크롬 등).
-   잠금이 활성화되면 조명 변화에 따른 급격한 색상 왜곡을 방지할 수 있습니다.

4. **큐브 방향 가이드 (Orientation Guide)**

솔버가 정확한 해답을 내놓으려면 각 면을 찍을 때 큐브의 **회전 방향**이 중요합니다. 화면의 안내에 따라 각 면의 중앙 색상을 맞추고, 동시에 '위쪽'에 오는 색상도 확인해 주세요.

| 촬영할 면 (중앙) | 큐브 위쪽에 와야 할 색상 |
| :--- | :--- |
| **흰색 (U)** | 파란색 (B) |
| **빨간색 (R)** | 흰색 (U) |
| **초록색 (F)** | 흰색 (U) |
| **노란색 (D)** | 초록색 (F) |
| **주황색 (L)** | 흰색 (U) |
| **파란색 (B)** | 흰색 (U) |

## 5. 문제 해결 (Troubleshooting)

-   **권한 오류**: 카메라 사용 권한이 거부된 경우 브라우저 설정에서 권한을 허용하고 '다시 시도' 버튼을 누르세요.
-   **장치 없음**: 카메라를 찾을 수 없는 경우 연결 상태를 확인하세요.
-   **인식 지연**: 조명이 너무 어두우면 프레임 노이즈로 인해 '안정성 체크'가 늦어질 수 있습니다. 더 밝은 곳에서 시도하세요.

## 6. 검증 및 테스트 (Verification)

카메라 제어 기능은 하드웨어 의존성이 높으므로 다음과 같은 항목을 수동으로 검증해야 합니다:

1.  **카메라 전환**: 버튼 클릭 시 `environment` ↔ `user` 모드가 즉시 반영되는지 확인.
2.  **스트림 재시작**: 전환 시 화면이 멈추지 않고 새로운 카메라 피드가 정상적으로 나오는지 확인.
3.  **권한 처리**: 카메라 전환 도중 브라우저가 권한을 재요청하거나 차단된 경우 적절한 에러 메시지가 표시되는지 확인.
4.  **성능**: 저사양 기기에서 전환 시 메모리 누수나 심한 지연이 없는지 확인.
