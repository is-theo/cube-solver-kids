# 🤝 기여 가이드 (Contributing Guide)

큐브 마법사 프로젝트에 관심을 가져주셔서 감사합니다! 이 문서는 원활한 개발 및 기여를 위한 가이드라인입니다.

## 🛠 개발 환경 설정

1.  저장소를 클론합니다.
2.  `npm install`로 의존성을 설치합니다.
3.  `npm run dev`로 개발 서버를 실행합니다.

## 📜 코딩 스타일 및 원칙

*   **TypeScript**: 모든 코드는 타입을 명시하며, `any` 사용을 지양합니다.
*   **React**: 함수형 컴포넌트와 Hook을 사용합니다. 성능 최적화(rAF, memo 등)가 필요한 부분은 신중하게 적용합니다.
*   **색상 인식**: `src/lib/colorDetector.ts`의 로직을 수정할 경우, 다양한 조명 환경을 고려해야 합니다.

## 🧪 테스트 (Testing)

새로운 기능을 추가하거나 버그를 수정할 때 반드시 테스트를 수행해주세요.

*   `npm test`: 전체 유닛 테스트 실행.
*   핵심 라이브러리(`src/lib/*.ts`) 수정 시 대응하는 테스트 파일(`*.test.ts`)도 함께 업데이트해야 합니다.

## 🚀 풀 리퀘스트 (PR) 절차

1.  새로운 브랜치를 생성합니다 (`feature/기능명` 또는 `fix/버그명`).
2.  작업을 완료하고 빌드(`npm run build`) 및 테스트가 통과하는지 확인합니다.
3.  변경 사항에 대한 명확한 설명을 포함하여 PR을 생성합니다.
